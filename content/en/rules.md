# Server Rules

## About This Server

This is a creative and casual server focused on building and community. The server takes an open stance toward client-side mods, but all in-game conduct must respect other players' creations and maintain order on the server.


## Player Ranks & Probation System

New players start as **Yellow** (probationary) members. During this period, admins will observe your behavior.

There is no fixed duration for probation — the system calculates your activity level and will automatically promote you to full member (**White / Member**) when the threshold is met. The exact algorithm is not disclosed. If any violations or inappropriate behavior occur during probation, admins may extend the probationary period.

- **Yellow (Probationary)**: The default rank for new players. The following restrictions apply:
  - Cannot use `/hdb` to access the decorative head database
  - Limited access to Axiom features
  - Cannot use server performance commands `/mspt` and `/tps`
  - Cannot use formatted text in chat
  - Cannot use `/rmitems` to clear nearby dropped items
  - Cannot use vanilla commands `/attribute`, `/damage`, `/effect`, `/fill`, `/item`, `/ride`, `/save-all`, `/setblock`
- **White (Member)**: Full membership granted automatically by the system based on activity. All restrictions above are lifted.


## Building & Creation Guidelines

1. **Do not grief or destroy other players' builds**, and do not damage terrain beyond what building requires. Examples of violations: digging random holes in the wilderness; slicing off half a mountain without smoothing it out.
2. **Builds should blend with their surroundings** — avoid jarring gaps or out-of-place structures, and choose your build location thoughtfully. Examples of violations: placing a massive concrete block structure in the middle of a dense forest with no transition; building a completely clashing style directly adjacent to existing builds in a dense area. Admins have final judgment on what is appropriate.
3. **Light redstone is allowed**, but large-scale or server-lagging redstone machines are prohibited. Keep high-frequency computations to a minimum. Hoppers generate constant performance overhead — avoid them where possible, and never stack large numbers of them. When in doubt, ask an admin first.
4. **Credit the builder and original creator (if applicable)** somewhere in your build in a reasonable location.


## Mod & Client Modification Policy

The server allows players to freely use client-side mods, including features typically considered "hacks" or "cheats." However, the following behaviors are **strictly prohibited** regardless of how they are achieved:

1. **Do not use mods to grief or modify other players' builds** — this includes moving, deleting, or overwriting blocks placed by others
2. **Do not maliciously disrupt other players' experience** — including force-teleporting, pushing, or killing others via mod features
3. **Do not use mods to bypass server restrictions** — such as forcing a change in another player's game mode or executing unauthorized commands
4. **Do not use any means to degrade server performance** — including spawning excessive entities or items, or triggering mass chunk loading

> In short: use mods however you like, as long as it doesn't affect others or harm the server.


## Conduct & Community Standards

1. **Be respectful** — hateful, insulting, harassing, or attacking speech toward other players in public channels is not allowed
2. **No personal attacks or deliberate provocation** — this includes malicious criticism of a player's build style, aesthetic choices, or personal identity
3. **No advertisements, referral links, or commercial promotions** unrelated to the server
4. **No spreading misinformation or malicious rumors** — especially false claims about server management or other players
